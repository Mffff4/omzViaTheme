# 🚀 omzViaTheme

Универсальный скрипт для автоматической настройки **Oh My Zsh** с темой **passion** и полезными плагинами. Поддерживает macOS и Linux (Ubuntu, Debian, Fedora и др.).

![Terminal Mockup](https://raw.githubusercontent.com/ChesterYue/ohmyzsh-theme-passion/main/demo/demo.png)

## ✨ Особенности

- 📦 **Автоматическая установка зависимостей**: zsh, git, curl, coreutils (на macOS).
- 🎨 **Тема Passion**: Минималистичная и информативная тема с поддержкой времени.
- 🔌 **Популярные плагины**:
  - `zsh-autosuggestions` — автодополнение команд.
  - `zsh-syntax-highlighting` — подсветка синтаксиса.
  - `sudo`, `git`, `history-substring-search` — встроенные в OMZ.
- 💻 **Кроссплатформенность**: Работает "из коробки" на macOS и Linux.
- 🛡️ **Безопасность**: Создает резервную копию вашего `.zshrc` перед изменениями.

## 🚀 Быстрая установка

Запустите следующую команду в вашем терминале:

```bash
curl -fsSL https://raw.githubusercontent.com/Mffff4/omzViaTheme/main/install.sh | bash
```

> [!IMPORTANT]
> На macOS скрипт **нельзя** запускать от пользователя `root`. Используйте обычного пользователя. Скрипт сам запросит `sudo`, когда это потребуется.

## 🛠️ Что делает скрипт?

1. Проверяет наличие `zsh`, `git`, `curl` и устанавливает их при необходимости.
2. Устанавливает **Oh My Zsh** (unattended mode).
3. Клонирует тему **passion** и необходимые плагины.
4. Настраивает `.zshrc`: устанавливает тему и список плагинов.
5. Меняет стандартный шелл на `zsh` (с проверкой пути на macOS).

## 🆘 Решение проблем

### Ошибка `gdate: command not found`

Эта ошибка может возникать на macOS при первом запуске, если тема уже активна, но `coreutils` еще не установлен. Скрипт установит `coreutils` автоматически, после чего ошибка исчезнет.

### Ошибка `The $ZSH folder already exists`

Если установка прервалась или вы хотите начать заново, выполните команду для полной очистки:

```bash
rm -rf ~/.oh-my-zsh ~/.zshrc
```

Затем запустите скрипт установки снова.
